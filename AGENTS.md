# RFCweb 项目协作规则

## 项目名称

RFCweb

## 项目目标

帮助一家做墙壁开关的公司完成官网搭建，并把页面真实落地到可访问的网站。

用户会先用 Google Stitch 完成初步网页风格设计，后续需要把设计转成真实网页，并以尽量低的成本上线。用户已经拥有自己的域名。

## 沟通规则

- 每次回答都要用用户能听懂的话解释，少用专业黑话。
- 如果必须使用技术词，要顺手解释它是什么意思。
- 先说结论，再说原因和下一步。
- 遇到不确定的需求，优先用问答方式梳理清楚，不要直接假设。
- 给方案时要说明大概成本、优缺点、适合什么情况。
- 不要只给抽象建议，要尽量给出可执行步骤。

## 文件与操作安全规则

- 不要乱删文件。
- 不要执行会清空、重置、覆盖大量内容的危险操作，除非用户明确要求并确认。
- 修改代码前，先查看现有文件结构和项目习惯。
- 只改和当前任务有关的文件，避免顺手重构无关内容。
- 如果发现用户已有改动，不要回退或覆盖，先理解并保留。
- 创建、修改、删除文件时，要在回答中说明做了什么。

## 官网搭建原则

- 优先控制成本，能用低价方案上线就不选复杂昂贵方案。
- 优先选择静态网站方案，例如 Cloudflare Pages、Vercel、Netlify 或 GitHub Pages。
- 如果只是企业展示官网，默认不引入昂贵服务器、数据库或复杂后台。
- 页面要适合墙壁开关公司：清晰展示产品、应用场景、公司实力、联系方式和询盘入口。
- 页面要能在手机和电脑上正常浏览。
- 网站要尽量加载快、结构清楚、方便后续维护。
- 如果需要表单，优先考虑低成本方案，例如表单服务、邮件转发、Cloudflare Turnstile 防垃圾提交等。

## 需求梳理问答方向

在正式搭建前，先帮助用户确认这些问题：

1. 官网主要目的：品牌展示、产品目录、获取询盘、招商代理，还是面向海外客户？
2. 目标客户：家装用户、工程商、经销商、外贸客户，还是 B2B 采购商？
3. 网站语言：中文、英文，还是中英双语？
4. 页面范围：首页、关于我们、产品中心、案例/应用场景、新闻、联系我们等。
5. 产品资料：是否已有产品图片、型号、参数、认证、价格区间、应用场景。
6. 品牌资料：公司介绍、工厂照片、资质证书、logo、品牌色、宣传语。
7. 询盘方式：电话、邮箱、WhatsApp、微信、表单、地图地址。
8. Google Stitch 输出：是图片、HTML、Figma 风格稿，还是其他格式。
9. 域名和上线：域名在哪个平台购买，是否方便修改 DNS。
10. 预算和维护：是否希望免费托管，是否能接受少量年费，后续谁更新内容。

## 推荐低成本方案思路

默认优先考虑：

1. 用 Google Stitch 的设计作为视觉参考。
2. 把设计落地成静态网站代码。
3. 使用 Git 管理项目文件。
4. 使用 Cloudflare Pages、Vercel、Netlify 或 GitHub Pages 免费/低价托管。
5. 把用户已有域名绑定到托管平台。
6. 配置 HTTPS，让网站可以通过安全链接访问。
7. 如需询盘表单，再接入低成本表单方案。

## 搭建 SOP 输出要求

当用户要求整理 SOP 时，需要包含：

- 准备资料清单
- 页面结构规划
- 设计稿交接方式
- 前端落地步骤
- 本地预览步骤
- 上线部署步骤
- 域名绑定步骤
- 表单和联系方式配置
- 基础 SEO 设置
- 后续维护方式
- 成本估算
- 常见问题和排查方法

## 回答风格

- 用中文回答。
- 语气清楚、耐心、直接。
- 不夸大，不乱承诺。
- 如果有多种路线，先推荐最省钱、最容易维护的一条。
- 每一步尽量解释“为什么这么做”。
